# Quality_control_pipeline

End-to-end pipeline for downloading public datasets from PRIDE,
converting proprietary formats to open mzML, and computing
ID-free quality control metrics exported as mzQC v1.0.0 files.

---

## Requirements

### Python (3.11 recommended)

```bash
/Library/Frameworks/Python.framework/Versions/3.11/bin/python3 -m pip install \
    scipy numpy pandas pyteomics lxml pymzml joblib tqdm requests pyopenms
```

`pyopenms` is optional but required for XIC-FWHM (MS:4000051) and XIC50
(MS:4000050). All other 59 metrics compute without it.

### For RAW file conversion (Step 3)

| Situation | Tool needed |
|---|---|
| Thermo `.raw` / `.RAW` | ThermoRawFileParser + Mono |
| Waters `.raw` | msconvert (Docker) |
| AB Sciex `.wiff` | msconvert (Windows only — not supported on Mac) |
| Bruker `.d` | msconvert (Docker) |
| Already `.mzML` / `.mzXML` / `.mgf` | Nothing — skip Step 3 |

#### Install Mono (Mac — required for ThermoRawFileParser)

```bash
# Download installer from https://www.mono-project.com/download/stable/
# After install, add to PATH:
echo 'export PATH="/Library/Frameworks/Mono.framework/Versions/Current/Commands:$PATH"' >> ~/.zshrc
source ~/.zshrc
mono --version   # should print: Mono JIT compiler version 6.12.x
```

#### Install ThermoRawFileParser

1. Go to `https://github.com/compomics/ThermoRawFileParser/releases/tag/v1.4.5`
2. Under **Assets**, click **ThermoRawFileParser1.4.5.zip** (3.43 MB)
3. Unzip into the pipeline folder:

```bash
cd /path/to/QC_pipeline
unzip ~/Downloads/ThermoRawFileParser1.4.5.zip -d ThermoRawFileParser
mono ThermoRawFileParser/ThermoRawFileParser.exe --version   # should print 1.4.5
```

#### Install Docker (for Waters / Bruker conversion)

Download Docker Desktop from `https://www.docker.com/products/docker-desktop/`

> **Note:** Docker with Wine fails on Apple Silicon Macs (M1/M2/M3).
> Use ThermoRawFileParser + Mono for Thermo files instead.

## Step-by-step guide

### Step 1 — Search PRIDE and build dataset list

```bash
python3 data.py
```

Queries the PRIDE REST API (v2), searches for recent datasets with small
mzML or RAW files (< 150 MB each), and saves a list of up to 15 datasets
to `pride_datasets.json`.

**Output:** `pride_datasets.json`

---

### Step 2 — Download files from PRIDE

```bash
python3 download.py
```

- **mzML datasets** are downloaded directly into `data/<ACCESSION>/`
  and are immediately ready for the pipeline.
- **RAW datasets** are downloaded into `data_raw/<ACCESSION>/`
  and must be converted first (Step 3).

Downloads use FTP with automatic HTTPS fallback.
Files larger than 150 MB are skipped. Up to 5 files per dataset are downloaded.

**Output:**
```
data/
    PXD073126/  *.mzML
    PXD075396/  *.mzML
    ...
data_raw/
    PXD077492/  *.raw
    PXD077765/  *.raw
    ...
```

---

### Step 3 — Convert RAW files to mzML (only if needed)

Skip this step if all your datasets already have mzML files.

#### 3A — Thermo `.raw` / `.RAW` files

```bash
cd /path/to/QC_pipeline

mkdir -p data/PXD077492
mono ThermoRawFileParser/ThermoRawFileParser.exe \
    -d=data_raw/PXD077492 \
    -o=data/PXD077492 \
    -f=1          # 1 = mzML output

mkdir -p data/PXD077765
mono ThermoRawFileParser/ThermoRawFileParser.exe \
    -d=data_raw/PXD077765 \
    -o=data/PXD077765 \
    -f=1

mkdir -p data/PXD078272
mono ThermoRawFileParser/ThermoRawFileParser.exe \
    -d=data_raw/PXD078272 \
    -o=data/PXD078272 \
    -f=1
```

Or use the provided script (tries Docker first, then direct install):

```bash
python3 convert.py
```

#### 3B — MGF files mislabelled as mzML

Some PRIDE datasets publish MGF files with a `.mzML` extension.
The pipeline will crash with `XMLSyntaxError: Start tag expected` on these.

**Diagnosis:**
```bash
head -c 50 data/PXD073407/ORAN_SPERM_3-1-2.mzML
# If you see:  BEGIN IONS   ← it is actually an MGF file
```

**Fix — rename to `.mgf` (recommended):**
```bash
cd data/PXD073407
for f in *.mzML; do mv "$f" "${f%.mzML}.mgf"; done
```

The pipeline reads `.mgf` files natively. It automatically prefers MGF over
any same-name `.mzML` in the same folder (deduplication by stem), so you
do not need to delete old files if both exist.

> **Do not manually convert MGF to mzML.**
> A binary encoding bug in our earlier conversion script produced mzML files
> with wrong cvParam accessions, causing
> `ValueError: buffer size must be a multiple of element size`.
> The original MGF files work correctly and produce identical metrics.

#### 3C — mzXML files

No conversion needed. The pipeline reads `.mzXML` natively.

#### 3D — AB Sciex `.wiff` files

Not supported on Mac or Linux. Move them to `data_raw/`:

```bash
mv data/PXD078909 data_raw/PXD078909
```

The pipeline silently skips folders with no recognised MS files and
reports them as: `Skipped (no MS files): PXD078909`.

---

### Step 4 — Run the QC pipeline

```bash
cd /path/to/QC_pipeline
/Library/Frameworks/Python.framework/Versions/3.11/bin/python3 run_pipeline.py
```

Or if `python3` already points to 3.11:

```bash
python3 run_pipeline.py
```

The pipeline will:

1. Find all `.mzML`, `.mzXML`, and `.mgf` files under `data/`
2. **Deduplicate** — if a folder has both `file.mgf` and `file.mzML` with
   the same stem, only the MGF is processed (MGF > mzML > mzXML priority)
3. Skip any subfolder with no recognised MS files
4. Compute all 61 ID-free QC metrics for each file
5. Run XIC feature detection (pyopenms) on MS1 mzML files to compute
   XIC-FWHM (MS:4000051) and XIC50 (MS:4000050)
6. Write one `.mzQC` file (PSI v1.0.0) per input file
7. Write flat TSV with all metrics and a per-dataset summary TSV

**Expected runtime:** 2–15 seconds per file depending on size and
whether XIC feature detection runs.

## XIC feature detection (MS:4000051, MS:4000050)

### How it works

The pipeline runs three pyopenms algorithms on the MS1 data:

1. **MassTraceDetection** — finds connected regions of signal across
   consecutive scans at the same m/z (mass traces / XICs)
2. **ElutionPeakDetection** — splits each trace into individual
   chromatographic peaks and estimates their FWHM in seconds
3. **FeatureFindingMetabo** — groups isotope traces into features and
   returns RT, m/z, intensity, and FWHM for each detected peak

### Metrics produced

| Metric | CV ID | Definition |
|---|---|---|
| XIC-FWHM Q1/Q2/Q3 | MS:4000051 | Quantiles of chromatographic peak widths (minutes). Narrower = better separation. |
| XIC50 fraction | MS:4000050 | Fraction of features whose FWHM window captures ≥ 50% of total MS1 TIC. Higher = signal concentrated in peaks, not background. |

Both metrics apply to **both DDA and DIA** — feature detection runs on MS1
survey scans regardless of acquisition mode.

### When XIC metrics are N/A

| Condition | Result |
|---|---|
| pyopenms installed + mzML file + has MS1 | ✓ Computed |
| pyopenms not installed | N/A |
| File is mzXML or MGF | N/A — pyopenms reads only mzML |
| File has 0 MS1 scans (MS2-only) | N/A |
| Feature detection finds 0 features | N/A |

## Understanding N/A values

N/A values are **scientifically correct**, not pipeline errors.

| Value | Why it appears | Scientific meaning |
|---|---|---|
| MS1 metrics = N/A | File has 0 MS1 scans | MS2-only acquisition (IP-MS, targeted, TMT-MS2) |
| MS2 metrics = N/A | File has 0 MS2 scans | MS1-only survey (metabolomics, native MS) |
| Chrom dur = N/A | No MS1 signal above threshold | Cannot define active elution window |
| Metadata ok = 0 | Isolation window absent from file | Exported without isolation window coordinates |
| Mode = MS2-only | No MS1 present | DDA vs DIA cannot be determined |
| Mode = MS1-only | No MS2 present | Survey or metabolomics run |
| Mode = DDA-MS2only | No MS1, DDA-like precursors | DDA but MS1 not stored in file |
| Mode = DIA-MS2only | No MS1, repeating windows | DIA but MS1 not stored in file |
| DIA metrics = N/A | File is not DIA | Only computed for DIA and DIA-MS2only modes |
| XIC-FWHM = N/A | No MS1, or mzXML/MGF | Feature detection not possible |

## Supported file formats

| Format | Extension | How handled | XIC detection |
|---|---|---|---|
| mzML | `.mzML`, `.mzml` | Read natively (pyteomics) | ✓ Available |
| mzXML | `.mzXML`, `.mzxml` | Read natively (pyteomics) | N/A |
| MGF | `.mgf`, `.MGF` | Read natively (pyteomics) | N/A |
| Thermo RAW | `.raw`, `.RAW` | Convert with ThermoRawFileParser + Mono | — |
| Waters RAW | `.raw` (Waters) | Convert with msconvert (Docker) | — |
| AB Sciex WIFF | `.wiff` | Not supported on Mac/Linux | — |
| Bruker | `.d` | Convert with msconvert (Docker) | — |

### File deduplication

If a folder contains both `file.mgf` and `file.mzML` with the same stem,
the pipeline processes only one, in priority order: **MGF > mzML > mzXML**.
This prevents double-counting and avoids hand-converted mzML files that
may have binary encoding errors.

## Quick start (full pipeline from scratch)

```bash
# 1. Install dependencies
/Library/Frameworks/Python.framework/Versions/3.11/bin/python3 -m pip install \
    scipy numpy pandas pyteomics lxml requests pyopenms

# 2. Search PRIDE for datasets
python3 data.py

# 3. Download files
python3 download.py

# 4a. Convert Thermo RAW files
for acc in PXD077492 PXD077765 PXD078272; do
    mkdir -p data/$acc
    mono ThermoRawFileParser/ThermoRawFileParser.exe \
        -d=data_raw/$acc -o=data/$acc -f=1
done

# 4b. Handle MGF files mislabelled as mzML
cd data/PXD073407
for f in *.mzML; do mv "$f" "${f%.mzML}.mgf"; done
cd ../..

# 4c. Exclude unsupported formats
mv data/PXD078909 data_raw/PXD078909

# 5. Run QC pipeline
python3 run_pipeline.py

# Results:
#   results/tsv/all_metrics.tsv          — 61 metrics per file
#   results/tsv/summary_by_dataset.tsv   — 38 numeric metrics summarised
#   results/mzqc/<dataset>/<file>.mzQC   — mzQC v1.0.0 per file
```

## Troubleshooting

| Error | Cause | Fix |
|---|---|---|
| `ModuleNotFoundError: scipy` | Wrong Python version | Use full path: `/Library/Frameworks/Python.framework/Versions/3.11/bin/python3` |
| `XMLSyntaxError: Start tag expected` | File is MGF renamed to `.mzML` | `cd data/PXD073407 && for f in *.mzML; do mv "$f" "${f%.mzML}.mgf"; done` |
| `ValueError: buffer size must be a multiple of element size` | mzML hand-converted from MGF with wrong binary cvParams | Rename originals to `.mgf`; pipeline prefers MGF over same-name mzML |
| `zsh: command not found: mono` | Mono not in PATH | `echo 'export PATH="/Library/Frameworks/Mono.framework/Versions/Current/Commands:$PATH"' >> ~/.zshrc && source ~/.zshrc` |
| Docker `Assertion failed` / `core dumped` | Wine/QEMU fails on Apple Silicon | Use ThermoRawFileParser + Mono for Thermo files |
| `Skipped (no MS files): PXD078909` | Folder has only `.wiff` files | Expected — move to `data_raw/` |
| AUC MS1 = 0, Mode = MS2-only | File has no MS1 scans | Correct — common in TMT, IP-MS, targeted acquisitions |
| XIC-FWHM = N/A for all files | pyopenms not installed | `/Library/Frameworks/Python.framework/Versions/3.11/bin/python3 -m pip install pyopenms` |
| XIC-FWHM = N/A for mzXML / MGF | pyopenms reads only mzML | Expected — convert to mzML if XIC metrics are needed |
| `Determination of memory status is not supported` | pyopenms warning on macOS | Harmless — all metrics still computed correctly |
| DIA metrics = N/A for non-DIA files | Metrics only computed for DIA/DIA-MS2only | Correct — DIA windows cannot be measured in DDA or MS1-only files |
| Summary TSV missing DIA metrics | Only per-file TSV covers tuples | Summary intentionally excludes multi-value metrics; single-value DIA metrics (MS:4000193/194, NEW-016–020) are included |
